# スクロールバーを「問題ごとのメーター」にする仕様

## ゴール
- 各設問ごとに進捗メーターのような「メモリ」を表示する。
- **未回答の設問は色を変えない（＝グレーなど）**。
- メモリをタップすると、その設問の位置まで自動スクロールする。
- 質問数が 20 問前提（可変でも対応できる設計にする）。

---

## UI イメージ

- 画面右側（または上部）に縦（または横）のバーを配置。
- バーの上に「質問数分」のメモリ（小さなブロック or ドット）を等間隔で並べる。
- 各メモリは **1問 = 1メモリ** に対応。
- メモリの状態：
  - **未回答:** ベースカラー（例：#E0E0E0）
  - **回答済:** 強調カラー（例：#4CAF50）
  - **現在表示中の設問:** アクティブカラー（例：#FF9800）＋少し大きく

---

## 状態管理の前提

- **questions:** 質問配列（`id`, `title`, `answered` など）
- **currentQuestionIndex:** 現在表示中の設問インデックス
- **containerRef:** 質問リスト全体をラップする要素の ref（スクロール制御用）
- 各設問ブロックに `id="question-{index}"` のようなアンカーを付与しておく。

---

## コンポーネント構成案

### 1. QuestionList（質問本体）

- 各設問を `div` でラップし、`id="question-{index}"` を付与。
- 例：
  ```jsx
  <div ref={containerRef} className="question-list">
    {questions.map((q, index) => (
      <div
        key={q.id}
        id={`question-${index}`}
        className="question-block"
      >
        {/* 質問文・選択肢など */}
      </div>
    ))}
  </div>
