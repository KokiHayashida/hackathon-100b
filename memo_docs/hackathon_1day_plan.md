## 実装の優先度と手順

### 最優先：TypeDiagnose.jsx の改修

**Step 1：質問データの用意**
- `financial_type_questions.md` の 20問を `questions` 配列に変換
- 5段階質問：`{ id, text, type: 'scale', scale: [1,2,3,4,5] }` のような形式
- 分配質問：`{ id, text, type: 'allocation', options: [{ value:'A', label:'80億安定/20億成長', points: 0 }, ...] }` のような形式

**Step 2：診断ロジックの実装**
- `financial_type_logic.md` の計算式をそのまま JavaScript に移植
- `calculateScores(answers)` と `getTypeCode(scores)` を実装

**Step 3：16タイププロフィールの組み込み**
- `financial_type_profiles.md` から各タイプの name, 特徴, 長所, 短所 を `TYPE_PROFILES` に格納
- キーは4文字コード（例：`SLAP`, `RLIP`）

**Step 4：UI の調整**
- 20問なので、1ページに全表示 or ページネーション or ステップ形式のいずれか
- 時間がなければ、まずは1ページ全表示で完成度を優先

### 次点：RealityCompare.jsx の修正

27行目付近の `compare-big-n` と `umber` の誤った改行を修正し、`compare-big-number` に統一する。

### 余裕があれば

- SpendSpeed：プレゼンで触る「毎秒10万円」「毎日1000万円」をより強調
- ホーム画面：「20問で16タイプ」などの文言更新

## プレゼン資料の推奨構成（5〜8枚）

1. **表紙**：タイトル、チーム名、日付
2. **課題・背景**：「100億円をどう使うか」で性格が現れる / MBTI風診断の需要
3. **解決策**：100億円タイプ診断（FPTI）— 20問で16タイプ
4. **デモ**：実際のアプリ画面（診断→結果→消費スピードor現実比較）
5. **技術的ポイント**：4軸（S/R, L/N, A/I, P/C）、分配形式の質問、科学的根拠
6. **今後の展望**：レーダーチャート、AI連携、ブランド化など
7. **まとめ・質疑**：One-liner、連絡先など

ツール：Google スライド / PowerPoint / Canva など、共有しやすいものを使用。

## 時間が足りないときの割り切り案

1. **20問は後回し**：まず現行の3問のまま、4タイプ → 8タイプ程度に拡張してデモ可能にする
2. **16タイプは実装、質問は10問に短縮**：Q1, Q3, Q6, Q8, Q11, Q13, Q16, Q18 など、各軸2問ずつ
3. **プレゼンは4枚に圧縮**：表紙・課題・デモ・まとめ
4. **レーダーチャートは未実装でOK**：「今後の展望」でスライド上に説明するだけにする

完璧より「動くデモ＋伝わるプレゼン」を優先するのがおすすめです。
