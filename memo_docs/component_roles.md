# コンポーネント構成と役割まとめ

## 全体方針

- 画面ごとに 1 コンポーネントファイルを作り、役割をはっきり分ける。
- 画面の切り替えは `src/App.jsx` のタブで行い、それぞれの中身は専用コンポーネントに任せる。

## プロジェクト全体の説明

100億円の使い道から性格・価値観を診断したり、100億円のスケール感を体験するための React アプリです。

## ファイル一覧と役割

### `src/App.jsx`
- 役割: アプリ全体のレイアウトと画面切り替え（タブ）を担当する。
- やること:
  - `selectedView` state を持つ。
  - タブボタンを表示して、クリックで `selectedView` を変更する。
  - `selectedView` に応じて、下の 3 コンポーネントのどれかを表示する。
- 担当（例）: D さん（全体調整役）

### `src/components/TypeDiagnose.jsx`
- 役割: 100 億円タイプ診断（人生スキル診断）の画面。
- やること:
  - 質問リストと選択肢を定義する。
  - 回答を state で管理する。
  - 回答結果からタイプを判定するロジックを持つ。
  - 診断結果のタイトル・説明文・一言コメントを表示する。
- 担当（例）: A さん（診断ロジック担当）

### `src/components/SpendSpeed.jsx`
- 役割: 100 億円消費スピード体験の画面。
- やること:
  - 1 秒あたり or 1 日あたりの消費額を入力してもらう。
  - 100 億円が何日 / 何年で 0 になるかを計算する。
  - 結果を見やすい形で表示する（グラフは余裕があれば）。
- 担当（例）: B さん（計算ロジック担当）

### `src/components/RealityCompare.jsx`
- 役割: 現実比較モードの画面。
- やること:
  - 100 億円を「コンビニ何店舗分」「年収 500 万の人の生涯年収何人分」などに換算する。
  - 使用する金額の前提（例: コンビニ 1 店舗 = 〇万円など）をコメントか定数で残す。
- 担当（例）: C さん（調査・表示テキスト担当）

## 作業の進め方メモ

1. まず各ファイルに、h2 タイトルと簡単な説明だけのコンポーネントを用意する。
2. それぞれの担当者が、自分のファイルの中だけを中心に編集する。
3. 共通で使いたい UI が出てきたら、`src/components/common/` に切り出すことを検討する。
