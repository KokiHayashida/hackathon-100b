##1. メインコンセプトの変更
これまでの「100億円の使い道を計画するツール」から、**「100億円の使い道から性格を判定する『100億円タイプ診断』」**へとメイン機能を変更する。

旧案： 100億円をどう使うか設計・運用する実用・シミュレーション寄り。
新案： 質問に答えると「あなたは〇〇タイプ」と結果が出る、MBTIのようなエンタメ・診断寄り。


##2. コンセプト変更の背景と狙い
「診断」に振り切ることで、ユーザー体験をより楽しく、拡散性の高いものにする。

友達との比較： 「自分はこうだったけど、お前はどう？」と共有・比較したくなる心理を突く。
エンタメ性の追求： 実用性よりも、ゲーム感覚で楽しめる要素を重視。
ゴール設定の明確化： 「使い道を提示する」のではなく、「タイプが出る」ことをゴールにすることで、ユーザーのモチベーションを維持する。


##3. 具体的な機能・構成案
MBTI（16タイプ診断）の100億円版のようなイメージで構築する。
項目内容入力形式100億円の使い道に関する複数の質問に回答する形式。質問内容「何にいくら使うか」「どんな規模感で使うか」など、価値観が出る質問。出力結果回答パターンに基づいた「タイプ」を表示。サブ機能（余裕があれば）当初の「消費計画ツール」も残すが、優先度は「診断」が上。
###ユーザー体験（UX）の設計
フロー： 質問に答えていく（例：この商品を何個買うか等） → 最終的に「あなたは〇〇タイプ」という診断結果が出る。
メリット：
自分のタイプが出ることをゴールにすることで、利用のハードルが下がり、エンタメ性が増す。
タイプの種類を増やすことで、何度でも試したくなる。

ただし、診断機能（タイプ診断）に全力を注ぐため、他の機能は優先順位を下げ、必要最低限にする方向で調整。
市場性・広報
「MBTIが流行っているように、診断系はウケが良い」という認識。
広告収入などのビジネスモデルへの親和性も考慮。
他のチームとコンセプトが被らない、独自性の高い「エンタメ重視」の視点を強調する。
