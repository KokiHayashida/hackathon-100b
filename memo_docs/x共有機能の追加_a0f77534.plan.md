# X（Twitter）で結果を共有する機能の追加

## 概要

診断結果の `result-actions` 内に「Xで共有」ボタンを追加する。クリック時に X の Web Intent を開き、診断結果のテキストとアプリURLがプリセットされた状態で投稿画面を表示する。

---

## 技術仕様

**X Web Intent URL**: `https://twitter.com/intent/tweet`

| パラメータ | 用途 | 値の例 |
|------------|------|--------|
| text | ツイート本文（URL含め280文字以内） | 【100億円タイプ診断】私のタイプは「堅実ビルダー型」（SLAP）でした！ |
| url | 共有するページURL | 現在のページURL（`window.location.href`） |
| hashtags | ハッシュタグ（カンマ区切り、#なし） | 100億円タイプ診断,診断 |

- **text のみ**で完結させるか、**text + url** の両方を使うかは実装で選択可能
- URL を入れると text の文字数制限が実質 280 - 23 = 257 文字程度になる

---

## 実装内容

### 1. TypeDiagnose.jsx の変更

**handleShareToX 関数を追加**

```javascript
const handleShareToX = () => {
  if (!result) return
  const shareText = `【100億円タイプ診断】私のタイプは「${result.name}」（${result.typeCode}）でした！`
  const params = new URLSearchParams({
    text: shareText,
    url: window.location.href,
    hashtags: '100億円タイプ診断,診断',
  })
  window.open(`https://twitter.com/intent/tweet?${params.toString()}`, '_blank', 'noopener,noreferrer,width=550,height=420')
}
```

- `window.open` でポップアップまたは別タブで X の投稿画面を開く
- `noopener,noreferrer` でセキュリティを確保

**ボタンの追加**

- `result-actions` 内、「結果をコピー」ボタンの前か後に配置
- 例: `<button type="button" className="share-x-button" onClick={handleShareToX}>Xで共有</button>`

### 2. App.css のスタイル追加

- `.share-x-button`: 既存の `.copy-button` に近いスタイル
- X のブランドを連想させる色（黒系）や、既存 UI に合わせたデザイン

## 補足

- **ポップアップブロック**: 一部環境では `window.open` がブロックされる場合あり。その場合は `window.open` の代わりに `window.location.href = url` で同一タブ遷移にする案もある
- **via パラメータ**: アプリ用 X アカウントがあれば `via=アカウント名` でクレジットできる
- **テキストの調整**: feature の長文を含めると 280 文字を超えるため、短い 1 行コピーに留める
